🎙️ VoicePilot AI
The Problem

Existing voice assistants are opaque, cloud-dependent, and treat user data as a product. There was no privacy-first assistant with genuine long-term memory and extensible tool use.

The Solution

VoicePilot AI is a modular voice assistant with offline-first design. Wake-word triggers local speech recognition, which feeds into an LLM with a persistent vector memory store, enabling the assistant to recall context across sessions and execute tools on demand — all with explicit user permissions.

Tech Stack
PythonMediaPipeWhisperLangChainPorcupineVector DBOllamapyttsx3
Key Features
  • Custom wake-word detection — activates without continuous microphone processing
  • Offline-capable STT using Whisper for privacy-sensitive environments
  • Long-term memory via vector embeddings — context survives restarts
  • Extensible tool-calling framework — web search, calculator, system automation
  • Local LLM support via Ollama, with cloud fallback for complex reasoning
  • Explicit permission model — no action taken without instruction
Engineering Challenge

The hardest problem was designing a memory system that's both fast enough for real-time conversation and persistent enough to recall context from weeks ago. The solution uses vector similarity for retrieval and a summarization layer to compress old context without losing meaning.

💬 ChatFlow
The Problem

Modern teams and individuals need a real-time communication platform that combines instant messaging, group collaboration, media sharing, and AI assistance — all in one scalable, production-ready system.

The Solution

ChatFlow is a full-stack real-time messaging platform inspired by WhatsApp, Telegram, Discord, and Slack. It delivers instant messaging with Socket.IO, group conversations, media sharing via Cloudinary, AI-powered assistance, real-time notifications, typing indicators, read receipts, and an admin dashboard for platform analytics.

Tech Stack
Next.js 15React 19TypeScript Node.jsExpressSocket.IO PostgreSQLPrismaJWT Tailwind CSSCloudinaryDocker

  • Real-time messaging with Socket.IO — one-to-one and group conversations
  • Message editing, deletion, emoji reactions, typing indicators, and read receipts
  • Online presence detection and real-time notifications
  • Friend requests, group creation, member roles, and conversation history
  • Media sharing — image, video, and document uploads with Cloudinary storage

🛠️ ServiceCRM
The Problem

Field service businesses (HVAC, plumbing, electrical, landscaping) lack affordable, purpose-built CRM tools. They rely on spreadsheets, WhatsApp, and paper — leading to missed jobs, poor communication, no analytics, and no centralized customer management.

The Solution

ServiceCRM is a full multi-tenant SaaS CRM built with PHP and MySQL. It features role-based access control (RBAC) with Admin and Super Admin panels, job dispatch, technician tracking, customer portal, automated invoicing, and comprehensive analytics — all in a secure, production-grade platform.

Tech Stack
PHPMySQLHTML5 CSS3JavaScriptRBAC BootstrapjQueryAjax PDOMVC Architecture
Key Features
  • Multi-Tenant Architecture — Complete data isolation per business with database-level separation
  • RBAC (Role-Based Access Control) — Granular permissions for Super Admin, Admin, Manager, Technician, and Customer roles
  • Super Admin Panel — Platform-wide oversight, tenant management, system settings, and global analytics
  • Admin Panel — Business-specific dashboard, user management, job dispatch, and reporting
  • Job Dispatch Board — Real-time technician availability, smart routing, and status tracking
  • Customer Portal — Job status tracking, service history, feedback submission, and invoice viewing
  • Automated Invoicing — Dynamic invoice generation, PDF export, and payment tracking
  • Analytics Dashboard — Revenue metrics, job completion rates, technician performance, and customer satisfaction scores
  • Real-time Notifications — Email and in-app alerts for job assignments, status updates, and reminders
  • Secure Authentication — Password hashing, session management, and CSRF protection
Engineering Challenge

The biggest challenge was implementing true multi-tenancy with complete data isolation while maintaining performance and scalability. The solution uses a database-per-tenant approach with shared lookup tables, combined with a robust RBAC system that ensures users from different organizations never access each other's data — all while delivering sub-200ms response times for complex dashboard queries.
